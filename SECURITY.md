# Security Policy

## Reporting a vulnerability

Do not open a public GitHub issue for security-sensitive problems such as credential leaks, authentication bypasses, private key exposure, or supply-chain concerns.

Report security issues privately to the maintainers through an existing private contact channel. If GitHub Security Advisories are enabled for this repository, prefer that route.

## Response expectations

- Include clear reproduction details and impact.
- Share the smallest set of files, logs, or screenshots needed to verify the issue.
- Avoid publishing exploit details until the maintainers have had a reasonable chance to assess and fix the problem.
